# Ride Booking Analytics (SQL Project)

## Overview
This project analyzes a ride-booking dataset using SQL. The goal is to understand ride completion, revenue distribution, customer behavior, and operational inefficiencies.

## Dataset
The dataset contains ride-level data including:
- booking status
- payment method
- ride timestamp
- vehicle type
- ratings

## Workflow
1. Data Cleaning
2. Transformation
3. Final Table Creation
4. Validation
5. Feature Engineering
6. Views (Analysis Layer)
7. Insights Extraction

## Key Insights
- Ride completion rate is ~62%, with failures mainly due to driver-side issues
- UPI contributes ~45% of total revenue
- Peak demand occurs during evening hours (5 PM–8 PM)
- Revenue is evenly distributed across customers (top 10 ≈ 9%)
- Ratings are consistently high (4.0–4.6 range)
